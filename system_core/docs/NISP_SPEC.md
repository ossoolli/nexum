# 📜 Nexum Intent Specification Protocol (NISP) v1.0
## 🌌 Overview
NISP defines the standard structure for all "Intents" within the Nexum Sovereign OS. It ensures that requests are deterministic, measurable, and economically bound.
## 🏗️ Packet Structure (JSON Schema)
All packets sent to the Go Bridge must follow this high-density schema:

| Field | Type | Description |
| :--- | :--- | :--- |
| `intent_id` | UUID | Unique identifier generated by the Kernel. |
| `version` | String | Must be "v0.1". |
| `objective` | String | Semantic definition of the goal (e.g., `verify_document`). |
| `constraints` | Object | Limits: `max_cost`, `min_reputation`, `timeout`. |
| `reward` | Object | Amount in `NST` and escrow rules. |
| `metadata` | Object | Contextual data for the worker (e.g., `file_hash`). |

---
## 🛠️ Validation Rules for Go-Agent
1. **Schema Integrity:** Any packet missing a `reward` or `objective` must be rejected with Code 403 (Invalid Protocol).
2. **Deterministic Routing:** The Go Bridge must use the `min_reputation` constraint to filter workers before invoking the Matchmaker.
3. **Status Reporting:** Results must be wrapped in `---SOVEREIGN_START---` to prevent noise.
